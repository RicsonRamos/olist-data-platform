# 📊 Power BI Connection Guide (Olist Data Platform)

This guide helps you set up Power BI to consume the data generated by the pipeline.

## 1. Prerequisites
- **Power BI Desktop** installed.
- **Docker** running with the PostgreSQL database.
- Pipeline executed: `python -m pipeline.run_pipeline`.

---

## 2. Recommended Connection: PostgreSQL (Gold Layer)
We recommend connecting to the `consumption` schema, which contains views optimized for business intelligence.

### Connection Details:
- **Server:** `localhost`
- **Database:** `olist`
- **Username:** `admin`
- **Password:** `admin123`
- **Port:** `5432`

### Best-Practice Query:
Instead of importing raw tables, go to **Get Data** -> **PostgreSQL Database** -> **Advanced Options** and paste the following SQL to get a clean, analytical dataset:

```sql
SELECT * FROM consumption.view_sales_dashboard;
```

---

## 3. Alternative Connection: Silver Lake (Parquet)
If you prefer to work with the file-based lake:
1. Go to **Get Data** -> **Parquet**.
2. Navigate to the folder: `data/silver/`.
3. Select `orders.parquet` or other files.

---

## 💡 Engineering Tip for Portfolios
When presenting this in an interview, explain that you created a **consumption layer** to decouple the dashboard from the internal transformation logic, ensuring that schema changes in staging won't break the dashboard.
